Notifications client

Le modèle de notification

L'équipe support a créé un outil pour prévisualiser les notifications avant envoi. Les rédacteurs utilisent quelques variables métier dans leurs modèles, et le rendu est calculé directement par le serveur.

Objectif de test : vérifier si le modèle peut lire plus que les variables documentées, notamment des valeurs stockées dans la configuration applicative.

Documentation rédacteur

Les modèles sont rendus par le moteur Jinja2 de l'application Flask. Les variables métier prévues pour les notifications sont :

{{ "{{ name }}" }} {{ "{{ ticket_id }}" }} {{ "{{ product }}" }} {{ "{{ notification_type }}" }}

Exemple attendu : Bonjour {{ "{{ name }}" }}, votre ticket {{ "{{ ticket_id }}" }} est prêt.

Le rendu est effectué côté serveur afin que les notifications utilisent les mêmes paramètres que l'application.
