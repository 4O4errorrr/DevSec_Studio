Pourquoi ces profils existent

Un contexte pour chercher comme dans une vraie application

Les challenges utilisent plusieurs comptes, tickets, exports, notifications et profils pour montrer qu'une faille ne vit presque jamais toute seule. Une donnée visible par Alice peut parfois révéler quelque chose sur Bob ou Charlie ; une configuration laissée en ligne peut aider à comprendre une autre route ; une API pratique peut appliquer plus de champs que prévu.

L'objectif n'est pas de deviner au hasard, mais d'observer les différences entre les rôles, les écrans et les réponses serveur. C'est exactement ce qu'un test de sécurité cherche à provoquer avant la mise en production.
